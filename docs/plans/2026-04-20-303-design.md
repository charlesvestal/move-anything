# 303 Synth Module — Design

TB-303 emulator as an external Schwung sound generator. Pairs with `schwung-tb3po` (external sequencer) or any MIDI source.

## Goal

Port jc303's DSP (Open303 engine + Devilfish mods + GuitarML RNN overdrive) to a Schwung `sound_generator` module. No built-in sequencer — driven entirely by incoming MIDI.

## Non-goals

- No GUI, no JUCE — strip everything except the DSP core.
- No built-in sequencer; an external MIDI source drives it (tb3po, Move pads, external USB MIDI).
- No neural overdrive training; ship the 197 pre-trained models as-is.

## DSP source and licensing

Upstream: [midilab/jc303](https://github.com/midilab/jc303) (GPLv3). jc303 itself is a CMake+JUCE port of [Robin Schmidt's Open303](https://github.com/RobinSchmidt/Open303) (MIT for the Open303 engine core) with two additions:

1. **Devilfish-inspired mod params** — `normal_decay`, `accent_decay`, `feedback_hpf`, `soft_attack`, `slide_time`, `tanh_shaper_drive`. Pure DSP, baked into jc303's copy of Open303.
2. **GuitarML BYOD overdrive** — RTNeural-based RNN inference loading pre-trained pedal/amp captures from `.json` weight files. 37 models in `models/JC303/`, ~160 more in `models/Proteus_Tone_Packs/`. All bundled.

Module is GPLv3 (inherits from jc303). The Open303 engine core stays MIT.

## Architecture

```
                   ┌──────────────────────────────┐
  MIDI in  ──▶     │  Open303 (rosic)             │
  (note on/off,    │   - oscillator, filter       │
   velocity,       │   - envelope, accent, slide  │
   legato)         │   - Devilfish mods           │
                   └──────────┬───────────────────┘
                              │ float audio (mono)
                              ▼
                   ┌──────────────────────────────┐
                   │  GuitarML RNN overdrive      │
                   │   - RTNeural inference       │
                   │   - dry/wet mix              │
                   │   - bypassable               │
                   └──────────┬───────────────────┘
                              │
                              ▼ stereo int16 interleaved → plugin_api_v2 out
```

### MIDI convention

Matches tb3po and Open303's native expectations — zero translation:

- **Accent**: velocity ≥ ~100 (tb3po emits 118 for accent, 72 otherwise). Open303's `noteOn(key, vel)` applies accent internally based on velocity threshold.
- **Slide**: detected automatically by Open303 when a new note-on arrives while the previous note is still held (legato). tb3po withholds the prior note-off to trigger this.

No CC 65, no special sysex.

### Plugin API

`plugin_api_v2` — supports multiple instances (required for Signal Chain and shadow slots).

Per instance:
- `create_instance(module_dir, json_defaults)` — allocates `Open303`, `GuitarMLAmp`, loads default model from `module_dir/models/`.
- `on_midi(msg, len, source)` — routes note-on/off to `Open303::noteOn/Off`. Ignores other messages.
- `set_param(key, val)` / `get_param(key, buf, len)` — see parameter table below.
- `render_block(out_lr, frames)` — pulls mono from Open303, runs overdrive, interleaves to stereo int16.
- `destroy_instance` — frees both.

## Parameters

### Engine (Open303 core)

| Key | Type | Range | Notes |
|---|---|---|---|
| `cutoff` | float | 0..1 | Normalized to Open303's filter-cutoff range internally |
| `resonance` | float | 0..1 | |
| `env_mod` | float | 0..1 | Env-to-cutoff amount |
| `decay` | float | 0..1 | Env decay |
| `accent` | float | 0..1 | Accent amount |
| `volume` | float | 0..1 | |
| `waveform` | enum | saw / square | |
| `tuning` | float | 0..1 | Master tune (cents offset via Open303) |

### Devilfish mods

| Key | Type | Range |
|---|---|---|
| `normal_decay` | float | 0..1 |
| `accent_decay` | float | 0..1 |
| `feedback_hpf` | float | 0..1 |
| `soft_attack` | float | 0..1 |
| `slide_time` | float | 0..1 |
| `tanh_shaper_drive` | float | 0..1 |
| `devil_mod_switch` | enum | off / on |

### GuitarML overdrive

| Key | Type | Range |
|---|---|---|
| `overdrive_switch` | enum | off / on |
| `overdrive_level` | float | 0..1 |
| `overdrive_dry_wet` | float | 0..1 |
| `overdrive_model` | string | path relative to `models/`, e.g. `jc303/TS9_DriveKnob` |

`overdrive_model_list` (read-only) returns the list of available model names for the Shadow UI browser.

### Shadow UI hierarchy

```
root:
  knobs: [cutoff, resonance, env_mod, decay, accent, volume,
          overdrive_level, overdrive_dry_wet]
  params:
    - waveform
    - tuning
    - {level: "devilfish"}  → Devilfish mods submenu
    - {level: "overdrive"}  → Overdrive submenu (on/off, model picker)
```

`overdrive` submenu uses `items_param: overdrive_model_list` / `select_param: overdrive_model` for the 197-entry scrollable picker.

### Stripping JUCE

jc303's Open303 and GuitarML don't depend on JUCE at the DSP level — the coupling is in plugin-glue code (`JC303.cpp`, `JuceProcWrapper.h`, `BaseProcessor.h`, JUCE `BinaryData` for model embedding).

Our `plugin.cpp` replaces all of that:
- No `juce::AudioProcessor`, no value tree, no dry/wet mixer — use raw `float*` buffers and plugin_api_v2 `set_param`.
- Model loading: read `.json` files from `module_dir/models/` with `fopen`, parse with vendored nlohmann/json, feed weights to `RTNeural`. No `BinaryData`.
- `JuceProcWrapper` / `BaseProcessor` dropped; `GuitarMLAmp` wrapped directly.

## Build

`scripts/build.sh` (inside Docker with `aarch64-linux-gnu-g++`):

1. Compile Open303 sources → `.o`
2. Compile GuitarML sources (with `-I deps/RTNeural -I deps/json`) → `.o`
3. Compile `plugin.cpp` → `.o`
4. Link into `dsp.so` (C++17, `-O3`, no JUCE libs)
5. Stage `dist/303/` with `dsp.so`, `ui.js`, `module.json`, `help.json`, `models/`
6. `tar -czvf dist/303-module.tar.gz 303/`

Model files are copied from vendored `models/` sources into the tarball as-is.

Tag `v0.1.0`, GitHub Actions builds and uploads `303-module.tar.gz` + updates `release.json`. Then add an entry to `schwung/module-catalog.json`:

## Open questions

- **Tarball size**: 197 RNN model JSONs could be multi-MB. If the module tarball grows past a few MB, consider shipping a smaller default set (e.g. jc303-only, 37 models) and fetching the Proteus pack on demand.
- **CPU cost**: RTNeural on ARM64 is fast but not free. Measure the overdrive's load on Move's core-3 budget before shipping the more complex models by default.

Both deferred to implementation time.
